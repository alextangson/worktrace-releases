S-tello Trace 隐私政策

生效日期：2026 年 7 月 14 日 · 运营方：S-tello（S-tello Trace）

简要说明：S-tello Trace 以本地处理为默认。我们不出售个人数据，不投放基于工作内容的广告，也不会在用户不知情的情况下向雇主或第三方提供工作记录。

1. 适用范围

本政策说明 S-tello Trace macOS 与 Windows 客户端、公开下载与更新页面如何处理信息。S-tello Trace 用于把用户的工作活动整理为时间线、工作记忆和简报。

2. S-tello Trace 可能处理的信息

  • 工作活动：前台应用名称、窗口标题、开始与结束时间、空闲状态。
  • 浏览器信息：仅在用户授予相应系统权限后读取当前网页地址或相关上下文。
  • 屏幕内容：仅在用户授予屏幕录制权限并开启相关功能时，对屏幕图像进行本地截图、文字识别或视觉分析。
  • 日历与线下工作：仅在授权后读取所选日历；也可处理用户主动输入的会议、沟通、出差等线下工作。
  • 工作背景：用户主动提供的岗位、职责、关注方向及对自动分类结果的修正。
  • 配置与诊断：应用版本、功能设置、权限状态和本地运行错误。检查更新时，GitHub 等托管服务会按其正常网络服务流程收到 IP 地址、请求时间和客户端信息。

3. 信息如何使用

上述信息用于构建个人时间线、识别工作会话与工作流、生成简报、改善分类准确度、排查故障并提供软件更新。S-tello Trace 不使用工作内容进行广告画像，也不出售个人数据。

4. 本地存储、屏幕图像与保留

工作活动数据库和生成的记忆默认存储在用户设备上。启用屏幕理解时，屏幕图像可能在设备上被短期保存，用于 OCR、视觉分析和证据回看；应用会按照本地保留设置滚动清理。用户可关闭快照保留、暂停采集、排除敏感应用，或在应用中清除本地数据。

用户主动导出的文件由用户决定其保存位置、接收方与保留期限。卸载应用不一定会自动删除应用支持目录；如需完整删除，请参考支持页面或使用应用内的数据清除功能。

5. AI 服务与数据传输

S-tello Trace 支持本地 AI，也允许用户自行配置第三方 AI 服务。启用第三方 AI 后，为生成分类或简报所需的文本摘要、窗口信息、日历标题或其他工作信号可能被发送给用户选择的服务商；只有在用户明确开启相应视觉能力时，选定的图像才会发送。第三方服务将依据其自身条款与隐私政策处理数据。

API Key 和服务配置保存在用户设备上。用户可以不配置云端 AI，或改用受支持的本地模型。

6. 分享与披露

S-tello Trace 不会自动把个人工作记录提供给雇主。只有当用户主动导出、复制或发送简报时，所选内容才会交给相应接收方。为提供更新下载，GitHub 与 Microsoft 等分发平台会处理必要的网络和账户信息。若法律要求我们披露信息，我们只会在适用法律要求的范围内处理。

7. 用户控制

用户可以拒绝或撤销系统权限、暂停采集、设置应用排除规则、关闭屏幕快照、修正自动结果、删除本地数据并选择是否导出。撤销权限可能导致窗口标题、网页、日历或屏幕理解等功能不可用，但不会阻止用户查看已经保存在本地的数据。

8. 安全

我们采用与产品阶段和数据敏感性相称的技术与组织措施，包括本地优先存储、系统权限控制和减少不必要的数据传输。但任何系统都无法保证绝对安全，用户应保护设备登录凭据、API Key 与导出文件。

9. 儿童

S-tello Trace 面向专业工作场景，不面向 13 岁以下儿童，也不会故意收集 13 岁以下儿童的个人信息。

10. 政策更新

当产品能力或数据处理方式发生实质变化时，我们会更新本页面的生效日期，并通过版本更新说明或应用内入口提示用户。重大变更将在适用时重新征得用户确认。

11. 联系我们

如有隐私问题、数据删除请求或安全报告，请在 S-tello Trace Releases Issues 提交。请勿在公开 Issue 中粘贴屏幕截图、API Key、窗口标题或其他敏感工作内容；需要私下提供材料时，我们会给出安全的后续方式。

English summary

S-tello Trace is local-first. It processes work activity, optional browser context, calendar data, user-entered work context, and—only with system permission—screen content to build a personal timeline and work brief. Data is stored on the device by default. If you configure a third-party AI provider, the information needed for the selected analysis may be sent to that provider; images are sent only when the corresponding visual feature is explicitly enabled. S-tello Trace does not sell personal data or automatically disclose work records to employers. You can pause capture, revoke permissions, exclude sensitive apps, disable screenshot retention, delete local data, and control exports. For questions, use GitHub Issues without posting sensitive content.
